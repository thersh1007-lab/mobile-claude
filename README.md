# Mobile Claude

Chat with Claude AI from your phone with full access to your dev environment. Read files, edit code, run commands, approve tool calls, switch repos — all from your mobile browser.

A self-hosted mobile terminal for Claude that runs on your local network.

## Features

- **Two AI modes** — Direct API (streaming + tool approval) or Claude Code Bridge (full CLI with 20+ tools)
- **Session import** — Continue any Claude Code conversation on your phone
- **Tool approval** — See what Claude wants to do, approve or deny from your phone
- **Voice input** — Dictate messages via HTTPS + Whisper transcription
- **File browser** — Browse your entire codebase from your phone
- **Workspace switching** — Jump between repos without restarting
- **Chat history** — Conversations persist across restarts, browse and reload anytime
- **Desktop app** — Electron with tray icon, Ctrl+Shift+C toggle, auto-start
- **Self-hosted** — Your data stays on your machine, nothing leaves your network
- **43 integration tests** — Production-quality, not a toy

## Quick Start

```bash
git clone https://github.com/thersh1007-lab/mobile-claude.git
cd mobile-claude/server
npm install
npm run setup    # interactive wizard — sets up API key, auth token, certs
npm start        # starts server, shows QR code
```

Scan the QR code on your phone. Done.

## Or, Zero-Config Start

If you already have a `.env` file or want to configure via the browser:

```bash
cd mobile-claude/server
npm install
npm run build
npm start
```

Open the URL on any device — if no API key is configured, a setup wizard appears in the browser.

## Prerequisites

- **Node.js 18+**
- **Anthropic API key** — [console.anthropic.com](https://console.anthropic.com)
- (Optional) **Claude Code CLI** for Bridge mode — `npm install -g @anthropic-ai/claude-code`
- (Optional) **OpenAI API key** for voice transcription

## How It Works

```
Phone (PWA)  <──WebSocket──>  Node.js Server (your PC)  <──API──>  Claude
                                     |
                                File System
                              (your project files)
```

The server runs on your PC and serves a PWA to your phone over your local network. You chat with Claude, and when it wants to read a file or run a command, an approval card appears on your phone. You tap Approve or Deny.

In Claude Code Bridge mode, the server spawns the `claude` CLI for each message, giving you access to all 20+ Claude Code tools with streaming output and conversation memory.

## Configuration (.env)

| Variable | Required | Description |
|----------|----------|-------------|
| `ANTHROPIC_API_KEY` | Yes | Your Anthropic API key |
| `AUTH_TOKEN` | Yes | Secret token for auth (generated by setup) |
| `PORT` | No | Server port (default: 3456) |
| `WORKSPACE_ROOT` | No | Parent folder of your repos (auto-detected from git) |
| `OPENAI_API_KEY` | No | For Whisper voice transcription |
| `CLAUDE_PATH` | No | Path to claude CLI (auto-detected) |
| `AUTO_APPROVE` | No | `reads` (read-only tools) or `all` (everything) |
| `AUTO_APPROVE_TOOLS` | No | Comma-separated tool names to auto-approve |

## Remote Access

Install [Tailscale](https://tailscale.com/) on your PC and phone. Use the Tailscale IP instead of the local IP. Works from anywhere — no port forwarding.

## Security

- **LAN only** — server binds to `0.0.0.0` but is designed for local/Tailscale use
- **Token auth** — every WebSocket message requires the auth token
- **Rate limiting** — 5 failed auth attempts in 10 minutes = blocked
- **Path sandboxing** — file access restricted to workspace root
- **Command blocking** — `rm -rf`, `format`, `shutdown`, etc. are blocked
- **Audit logging** — all tool calls logged to `logs/audit.jsonl`
- **API keys stay server-side** — never sent to the client
